# Gotham 风格智能运营指挥台 UI 设计

> 日期：2026-05-20  
> 范围：说明如何借鉴 Palantir Gotham 式“事件驱动、态势画布、行动方案、时间轴”的 UI 结构，设计制造业智能运营指挥台。  
> 目标：让 ManuFoundry 的前台不只是后台表格，而是能展示风险、关系、方案和闭环的指挥调度界面。

## 1. Gotham 风格在这里代表什么

这里的 Gotham 风格不是军事题材，也不是照搬地图界面。

我们借鉴的是它的交互结构：

```text
左侧：事件 / 情报 / 方案列表
中间：地图或态势画布
右侧：方案详情 / 资源 / 审批 / 风险提示
底部：时间轴 / 计划 / 执行步骤
```

制造业版本应替换为：

```text
左侧：质量异常 / 供应链风险 / 设备预警 / 待处理任务
中间：业务图谱 / 生产拓扑 / 供应链影响路径
右侧：AI 建议 / 处置方案 / 业务动作 / 审批按钮
底部：处理时间线 / 工单进度 / 流程状态
```

## 2. 页面结构

```text
┌──────────────────────────────────────────────┐
│ 顶部：全局搜索 / 通知 / 当前角色 / 应用切换   │
├──────────────┬──────────────────┬────────────┤
│ 左侧事件栏    │ 中央态势画布       │ 右侧行动栏 │
│              │                  │            │
│ 高风险事件    │ 影响关系图         │ AI 分析    │
│ 处理中事件    │ 生产线拓扑         │ 处置方案   │
│ 我的任务      │ 供应链路径         │ 审批动作   │
│ 过滤器        │ 订单影响范围       │ 所需资源   │
├──────────────┴──────────────────┴────────────┤
│ 底部时间轴：发现异常 -> 分析 -> 派单 -> 审批 -> 闭环 │
└──────────────────────────────────────────────┘
```

## 3. 区域设计

### 3.1 顶部栏

职责：

- 全局搜索业务对象。
- 切换应用和角色上下文。
- 展示通知和待办数量。
- 进入账户中心。

搜索示例：

- 物料 `MAT-21`
- 工单 `WO-20260520-003`
- 设备 `EQ-03`
- 客户订单 `SO-7788`
- 供应商 `SUP-06`

### 3.2 左侧事件栏

职责：

- 展示当前角色需要关注的风险事件。
- 按严重度、状态、类型筛选。
- 支持快速切换事件。

示例：

```text
高风险
  3 号产线缺陷率异常
  MAT-21 物料库存低于安全线
  EQ-03 设备振动异常

处理中
  CAPA-20260520-001
  采购申请 PR-0088

我的任务
  2 个待审批
  1 个待补充资料
```

### 3.3 中央态势画布

职责：

- 展示当前事件的业务影响图。
- 支持点击对象节点查看详情。
- 支持高亮异常源头、受影响对象和已处理对象。

质量异常示例：

```text
供应商 SUP-06
   -> 物料 MAT-21
   -> 工单 WO-20260520-003
   -> 设备 EQ-03
   -> 质检 QC-8891
   -> 缺陷 D-102
   -> 客户订单 SO-7788
```

颜色建议：

| 状态 | 颜色 |
| --- | --- |
| 异常源头 | 红色 |
| 受影响对象 | 橙色 |
| 普通关联对象 | 蓝色或灰色 |
| 已处理对象 | 绿色 |
| 未确认风险 | 黄色 |

### 3.4 右侧行动栏

职责：

- 展示 AI 分析。
- 展示处置方案。
- 展示资源和影响范围。
- 提供可执行业务动作。

示例：

```text
AI 分析
  本次异常可能与 EQ-03 设备振动升高和 MAT-21 批次切换有关。

影响范围
  1 个工单
  1 个客户订单
  360 件同批库存

建议动作
  [生成质量整改任务]
  [冻结物料批次]
  [创建设备点检]
  [通知相关负责人]
```

### 3.5 底部时间轴

职责：

- 展示事件从发现到闭环的过程。
- 展示计划步骤和实际完成情况。
- 展示审批、评论和系统自动动作。

示例：

```text
09:18 发现异常
09:20 系统完成影响分析
09:25 质量经理确认风险
09:30 创建 CAPA 任务
10:10 设备点检完成
待完成 客户订单复检
```

## 4. 角色差异

同一个指挥台，应根据角色裁剪内容。

| 角色 | 左侧事件 | 中央画布 | 右侧动作 |
| --- | --- | --- | --- |
| 管理层 | 高风险事件、交付风险 | 全局影响图、客户订单影响 | 批示、催办、查看闭环 |
| 质量经理 | 质量异常、CAPA | 缺陷、质检、工单、物料关系 | 生成 CAPA、复检、冻结批次 |
| 生产主管 | 产线异常、工单风险 | 产线、设备、工单拓扑 | 暂停工单、调整计划 |
| 采购人员 | 供应商和物料风险 | 供应商、物料、库存关系 | 创建采购申请、供应商复核 |
| 设备工程师 | 设备预警、维修任务 | 设备、传感器、工单关系 | 接单、点检、维修反馈 |
| 审批人 | 待审批任务 | 关联对象摘要 | 同意、驳回、评论 |

## 5. 视觉风格

### 5.1 深色演示版

适合：

- 客户演示
- 大屏展示
- 管理层风险态势
- Palantir / Gotham 感较强的页面

建议：

- 深色背景。
- 蓝绿色关系线。
- 红橙色风险强调。
- 高对比信息卡。
- 中央画布优先。

### 5.2 浅色办公版

适合：

- 质量经理日常办公。
- 审批人处理任务。
- 管理员配置系统。

建议：

- 保持 Ant Design 企业风格。
- 信息密度更高。
- 表单、表格、审批评论更清晰。
- 不追求大屏酷炫。

## 6. MVP 页面建议

第一版“智能运营指挥台”只做一个质量异常场景即可：

1. 左侧固定展示 3 到 5 个风险事件。
2. 中间展示一张质量异常影响图谱。
3. 点击节点时，右侧显示对象详情。
4. 右侧展示 AI 分析和 4 个动作按钮。
5. 底部展示处理时间轴。
6. 点击“生成质量整改任务”弹出 CAPA 表单。
7. 提交后时间轴新增一条处理记录。

## 7. 与现有模块映射

| UI 能力 | 当前模块 | 建议 |
| --- | --- | --- |
| 风险事件队列 | Dashboard / Quality / Rules | 新增统一 RiskEvent 视图 |
| 中央图谱 | GraphExplorer / Cytoscape | 嵌入业务详情页，不只做独立工具 |
| AI 行动栏 | AIAssistant / AiChatWidget | 改成上下文感知 AI 面板 |
| 动作按钮 | Workflow / Notifications / Rules | 封装成业务 Action |
| 时间轴 | Workflow / Audit | 合并流程记录和操作日志 |

## 8. 演示话术

```text
这不是一个普通报表页。
普通报表只能告诉我们缺陷率升高了。
这个指挥台会告诉我们：
问题来自哪里，影响了哪些物料、设备、工单和客户订单，
AI 建议怎么处理，
以及能不能一键把处理任务派出去并持续追踪闭环。
```
